# Веб-приложение "Меню для кафе и ресторанов доставки"

Интерактивное веб-приложение (SPA) для просмотра меню, оформления заказов и взаимодействия с клиентами. Проект разработан в рамках учебной практики и представляет собой клиент-серверное решение с разделением логики на фронтенд и бэкенд.

## 🛠 Стек технологий

- **Frontend:** React, Mantine, Redux Toolkit, Vite, i18next.
- **Backend:** Node.js, Fastify, JWT, Bcrypt.
- **Database / Infrastructure:** PostgreSQL, пул соединений pg.Pool, Docker Compose.
- **Инструменты качества кода:** ESLint.

---

## ⚙️ Настройка переменных окружения (.env)

Для корректной работы пула соединений PostgreSQL и бэкенда, создайте файл `.env` в директории `./backend/docker/` (или скопируйте значения из примера):

```env
PG_USER=postgres
PG_PASS=your_secure_password
PG_DB=hexly_edu_db
PG_DATA=/var/lib/pgsql/data
PG_PORT=5432
JWT_SECRET=your_super_secret_jwt_key
```

## 🚀 Быстрый запуск (Режим разработки)

Все основные команды автоматизированы с помощью `Makefile`. 

> **Важно:** Для фонового запуска процессов frontend и backend используются утилиты `nohup` и `pkill`. Логи сервера и клиента записываются в директорию `./log/`. Перед запуском убедитесь, что папка `./log` создана.

### Пошаговая инструкция по запуску:

1. **Клонируйте репозиторий и установите зависимости:**
   ```bash
   npm install
   ```
2. **Настройте переменные окружения**, как описано в разделе выше.
3. **Запустите всю среду разработки:**
   Команда автоматически поднимет базу данных в Docker, запустит бэкенд и фронтенд в фоновом режиме:
   ```bash
   make run-dev
   ```
   *После успешного запуска логи работы приложений можно найти в файлах `./log/backend.log` и `./log/frontend.log`.*

4. **Остановка среды разработки:**
   Команда остановит контейнер с базой данных и корректно завершит фоновые процессы Node.js и Vite:
   ```bash
   make stop-dev
   ```

## 📋 Все доступные команды (Управление по отдельности)

Если вам необходимо управлять компонентами системы по отдельности, используйте следующие команды:

### Управление базой данных (Docker)
*   `make start-db` — Запустить контейнер с базой данных в фоновом режиме.
*   `make stop-db` — Остановить контейнер базы данных.
*   `make remove-db-data` — Полностью удалить локальные данные базы данных (очистка директории `data/`). **Внимание: данные будут утеряны!**

### Управление приложениями
*   `make start-frontend` — Запустить локальный сервер разработки фронтенда (Vite).
*   `make start-backend` — Запустить сервер бэкенда (`node backend/server.js`).

## 🌐 Спецификация REST API (v1)

Все эндпоинты бэкенда используют базовый префикс `/api/v1`. Логика работы с базой данных изолирована в слое данных (`queries.js`).

### Публичные эндпоинты (Доступны всем)
*   `GET /` — Проверка работоспособности сервера (`{"hello": "world"}`).
*   `GET /api/v1/dishes` — Получение списка всех блюд из оптимизированного SQL-представления `dish_cards`.
*   `POST /api/v1/auth/register` — Регистрация нового пользователя. Хэширует пароль через `bcrypt` и возвращает авторизационный JWT-токен.
*   `POST /api/v1/auth/login` — Аутентификация пользователя. Проверяет корректность данных и возвращает токен доступа (`expiresIn: '24h'`).

### Приватные эндпоинты (Требуют заголовок `Authorization: Bearer <token>`)
*   `PUT /api/v1/auth/update-account` — Динамическое обновление разрешенных колонок профиля (`user_name`, `email`, `phone`). Включает встроенную защиту от SQL-инъекций и проверку занятости email.
*   `DELETE /api/v1/auth/delete-account` — Полное и безвозвратное удаление аккаунта текущего пользователя из базы данных.

## 💡 Функциональные возможности приложения

1. **Каталог и Меню:** Два режима отображения. Быстрое переключение меню между адаптивной сеткой карточек и компактной таблицей.
2. **Авторизация и Регистрация:** Полноценная аутентификация пользователей с валидацией форм.
3. **Управление профилем:** Редактирование разрешенных колонок профиля. Удаление профиля пользователя.
4. **Корзина:** Управление заказами, изменение количества блюд, автоматический расчёт стоимости.
5. **Раздел «Помощь»:** Обратная связь со службой поддержки, история обращений в виде таблицы.
6. **Синхронизация с БД:** Все данные сохраняются на сервере в СУБД PostgreSQL.
