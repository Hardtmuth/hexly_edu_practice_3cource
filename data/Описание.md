# Описание проекта для учебной практики
## Тема проекта: Разработка веб-приложения «Меню для кафе и ресторанов доставки»

### 1. Актуальность темы

В условиях высокой конкуренции в сфере общественного питания автоматизация процессов взаимодействия с клиентами становится ключевым фактором успеха. Традиционные бумажные меню уступают место интерактивным цифровым решениям. Разработка веб-приложения для онлайн-заказа позволяет заведениям:

  - Сократить время на обработку заказов и снизить нагрузку на персонал
  - Предоставить пользователям удобный и наглядный инструмент для выбора блюд в режиме реального времени
  - Быстро обновлять ассортимент, цены и информацию об акциях без затрат на полиграфию
  - Повысить лояльность клиентов за счёт создания персонализированного пользовательского опыта и прозрачной системы обратной связи

### 2. Целевая аудитория (ЦА)

Проект ориентирован на две основные группы пользователей:

  - **Конечные потребители (клиенты)**: Люди разного возраста, предпочитающие заказывать еду на дом или в офис, ценящие скорость, удобный интерфейс, информативность меню и наличие личного кабинета для отслеживания заказов
  - **Администраторы и менеджеры заведений**: Персонал кафе/ресторана, которому необходим удобный инструмент для управления каталогом товаров, изменения статусов заказов и оперативной обработки обращений граждан

### 3. Предполагаемые функциональные возможности

Приложение разделено на клиентскую часть и личный кабинет пользователя. Основной функционал включает:

  - **Каталог и Меню**:

    - **Глобальная навигация**: Реализована система фильтрации блюд по категориям («Завтрак», «Обед», «Ужин», «Напитки», «Бизнес-ланч»)
    - **Управление состоянием через URL**: Фильтрация работает на основе параметров строки поиска `/?category=...`. Это обеспечивает сохранение выбранных фильтров при перезагрузке страницы и позволяет делиться прямыми ссылками на конкретные разделы меню
    - **Переключение видов**: Реализована возможность отображения меню в двух вариантах: адаптивная сетка карточек (*RenderCard*) с крупными изображениями блюд для визуального выбора и компактный табличный вид (*RenderTable*) для быстрого ориентирования в больших категориях
    - **Асинхронная индикация (UX)**: Кнопки добавления товара в корзину оснащены стейт-индикаторами. При клике активируется состояние загрузки (*эффект loading: dots*), после чего интерфейс выводит анимированное подтверждение об успешном добавлении (*btnAdded / иконка корзины*), предотвращая повторные случайные клики
    - **Детальный просмотр**: Модальное окно карточки блюда

  - **Авторизация и профиль**: Регистрация и аутентификация пользователей, хранение персональных данных в профиле

  - **Личный кабинет клиента**:

    - **Просмотр истории заказов**:и их текущего статуса («В обработке», «Доставляется», «Выполнен»)
    - **Управление адресами доставки**: Возможность сохранения нескольких адресов (например, «Дом», «Работа») для быстрого оформления последующих заказов
    - **Скидки, бонусы и акции**: Отображение текущего уровня персональной скидки пользователя, баланса бонусных баллов и списка доступных индивидуальных промокодов и акций заведения
    - **Раздел «Помощь»**: Интерактивная форма отправки обращений в службу поддержки и отображением истории обращений

  - **Безопасное управление профилем (CRUD)**:

    - Возможность для авторизованных пользователей динамически обновлять свои персональные данные (имя, email, телефон) с автоматической проверкой на уникальность email в базе данных
    - Функция полного удаления аккаунта пользователем из системы по

### 4. Используемые технологии

Для реализации клиентской части приложения выбран современный стек фронтенд-разработки:

- **Фронтенд (Клиентская часть)**:

  - **Язык программирования**: JavaScript (ES6+)
  - **Библиотека интерфейса**: React (компонентная архитектура, SPA)
  - **Управление состоянием**: Redux Toolkit (централизованный глобальный стейт)
  - **UI-фреймворк**: Mantine v7 (готовые компоненты, хуки и валидация форм)
  - **Графика**: Tabler Icons (набор векторных иконок)
  - **Локализация**: i18next (поддержка мультиязычности интерфейса)

- **Бэкенд (Серверная часть)**:

  - **Серверная платформа**: Node.js + Fastify
  - **Архитектура**: REST API с разделением на слой маршрутов и слой запросов

- **База данных и инфраструктура**:

  - **СУБД**: PostgreSQL (реляционная база данных)
  - **Оптимизация**: Слой доступа pg.Pool (пул соединений) и SQL-представления (Views)
  - **Контейнеризация**: Docker Compose (изолированное развертывание БД в dev-окружении)

- **Безопасность**:

  - **Аутентификация**: JSON Web Tokens (JWT) - авторизация эндпоинтов через Bearer-токены
  - **Криптография**: Bcrypt - асинхронное хэширование паролей с солью перед сохранением в БД

### 5. Ожидаемый результат

Итогом учебной практики является работоспособное клиент-серверное веб-приложение (SPA) с адаптивным дизайном.В рамках проекта реализованы и интегрированы с бэкендом следующие модули:

  - **Система аутентификации**: Валидируемые формы регистрации и авторизации пользователей с отправкой данных в базу данных и сохранением сессии
  - **Функционал корзины**: Динамическое добавление, удаление и изменение количества блюд с автоматическим пересчётом итоговой стоимости заказа
  - **Управление профилем**: Интерфейс добавления, редактирования и сохранения информации о пользователе и адресов доставки пользователя
  - **Обратная связь**: Валидируемая форма отправки обращений в поддержку с динамическим обновлением таблицы обращений в личном кабинете
  - **Интеграция с бэкендом**: Настроено взаимодействие с REST API, благодаря чему все данные пользователей сохраняются и обрабатываются в базе данных
